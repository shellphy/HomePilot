# HomePilot · 天青府家园

服务单个小区（招商天青府，约 600 户）的微信小程序。它把业主群里散落、说过就沉底的表态，沉淀成可行动、可追溯的集体事实。装修团购是它跑通的第一个场景。

## 核心模型

一切皆**事项（Matter）**：业主发起，管理员审核后公示，其他业主参与，结果随生命周期沉淀。参与有三种方式——**登记**（事前意向）、**接龙**（事中承诺）、**评价**（事后反馈）。每个已公示的事项都带「大家都在问」，业主就地提问，大模型结合小区上下文作答。

新场景由「事项类型 + 参与方式 + 公示模板」组合而成，多数靠配置落地。

## 七类事项与各自的流程

- **团购**：意向征集 → 谈判中 → 接龙中 → 已成团。发起人即团长；问答里的好答案可沉淀成「买前必懂」词条；成团后团长与确认参团的邻居互通联系方式；结束后参与者评价团长。
- **闲置**：挂价、传图、上架（在售）→ 有意向的邻居报名 → 与卖家互通联系方式 → 已出手。
- **活动**：见面会、组团踩点、球局——报名中 → 已结束，报名即参加。
- **互助**：拼车、借物、代收、搭伙看工地——进行中 → 已结束，凑人即成。
- **维权**：对开发商 / 物业集体发声——联名征集 → 交涉中 → 已有结果，联名即力量。
- **征集**：面向全小区收集结构化问卷表态，结果匿名聚合公示——征集中 → 已结束。
- **公告**：单向公示——已发布 → 已归档。

## 身份与权限

- **业主**：绑定楼栋号，可发起类型允许的事项，并参与全部事项。
- **相关方**：商家 / 物业 / 业委会 / 开发商自助入驻或经管理员核验后公示；治理类相关方能在事项时间线上发布官方回应。
- **管理员**专注审核：事项与相关方过审、拉黑违规成员，按 ownership 授权参与业务。

## AI 与内容安全

- **AI 起草**：发起事项、拟词条时由大模型打草稿，可联网检索。
- **AI 答疑**：事项详情里就地提问，结合小区上下文作答。
- **内容安全**：业主提交的文本经微信内容安全检测，命中违规即拦在入库前。

## 技术栈

- **前端**：原生微信小程序 + TDesign（`miniprogram/`），管理端也在小程序内
- **后端**：Laravel 13 + Sanctum + SQLite（`app/` `routes/` `database/`）
- **AI**：laravel/ai（DeepSeek）
- **部署**：Docker 单容器 FrankenPHP（Caddy 内嵌 PHP + 自动 HTTPS），独立 worker 跑队列
- **测试**：Pest（后端主战场；小程序界面在微信开发者工具人工验证）

## 目录

```
miniprogram/    微信小程序（pages/ components/ utils/）
app/            Laravel 领域模型与控制器（Matters/ Models/ Http/）
routes/api.php  小程序调用的 API
database/       迁移与种子
tests/          Pest 测试
```

## 本地开发

```bash
composer install && npm install
cp .env.example .env && php artisan key:generate
php artisan migrate --seed
composer run dev          # 起后端 + vite
php artisan test          # 跑测试
```

小程序端用微信开发者工具打开 `miniprogram/`，在 `.env` 配好 `WECHAT_APPID/SECRET`，工具里勾选「不校验合法域名」联调。
